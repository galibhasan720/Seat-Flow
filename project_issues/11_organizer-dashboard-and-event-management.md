## Organizer Dashboard and Event Management

**Task**
Provide organizers with the tools to create, update, publish, and monitor events plus the analytics needed to manage attendance and seat capacity.

**Description**
Implement organizer-only routes and backend APIs for event CRUD, booking-window control, capacity configuration, attendee lists, and dashboard summaries. The dashboard should expose the core metrics from the MVP scope, including seats sold versus available, booking trends, and event-level performance, while keeping the organizer workflow simple enough for a learning project.

**Subtasks**
- [ ] Build organizer event create/edit/publish flows.
- [ ] Add booking-window open/close controls.
- [ ] Implement seat-capacity and category configuration for events.
- [ ] Build attendee list and event summary views.
- [ ] Add organizer analytics cards and trend visualizations.

**Acceptance Criteria**
- [ ] Organizer accounts can manage only their permitted events.
- [ ] Event creation and updates persist the expected metadata and capacity rules.
- [ ] Attendee and analytics views reflect booking data accurately.
- [ ] Booking window control affects whether new reservations can be created.

**Deliverables**
* Organizer dashboard UI.
* Event management API coverage.
* Analytics and attendee list views.
