Project Name: The Multi-Channel Customer Intelligence Hub.   System Type: Designed as an enterprise-grade Customer Data Platform (CDP).   Core Function: The system aggregates heterogeneous customer interactions into a unified timeline.   Data Sources: It tracks varied interactions, including web clicks, support tickets, and purchase history.   Backend Architecture: Orchestrated by a layered Python FastAPI backend using Controller, Service, and Repository layers.   Database Architecture: Implements an advanced Single-Table NoSQL design via Amazon DynamoDB using an Event Sourcing pattern.   Frontend Constraints: The administrative frontend is explicitly constrained to a desktop-optimized viewport.   UI Framework: Utilizes component-driven frameworks to rapidly render complex data visualizations.
